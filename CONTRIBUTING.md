# Guía para contribuir

¡Gracias por tu interés en contribuir a este repositorio!

## ¿Cómo puedes contribuir?

### 1. Reportar errores o información incorrecta

Si encuentras información errónea o desactualizada en el informe:

- Abre un **issue** describiendo el problema.
- Proporciona fuentes o referencias que respalden tu corrección.
- Espera la revisión y respuesta del mantenedor.

### 2. Sugerir mejoras

Si tienes sugerencias para mejorar el contenido o la estructura:

- Abre un **issue** con la etiqueta `enhancement`.
- Describe claramente tu propuesta y el motivo.

### 3. Correcciones menores

Para correcciones tipográficas, gramaticales o de formato:

- Puedes abrir un **pull request** directamente.
- Asegúrate de que el cambio sea pequeño y bien justificado.

## Proceso de contribución

1. **Fork** el repositorio.
2. Crea una **rama** para tu cambio (`git checkout -b fix/descripcion`).
3. Realiza tus cambios y haz **commit** (`git commit -m "Descripción del cambio"`).
4. **Push** a tu rama (`git push origin fix/descripcion`).
5. Abre un **pull request** describiendo tus cambios.

## Estilo y formato

- Utiliza Markdown para todos los documentos.
- Mantén un tono profesional pero accesible.
- Asegúrate de que las tablas sean legibles en formato Markdown.
- Respeta la estructura existente del informe.

## Revisión

Todos los pull requests serán revisados por el mantenedor. Podemos solicitar cambios adicionales antes de aceptar la contribución.

## Código de conducta

Al contribuir, aceptas seguir nuestro [Código de Conducta](CODE_OF_CONDUCT.md).
